For this, I experimented with some multi-agent workflows. I couldn't find any previous problems I thought fit very well, so I made some new ones up.

The first workflow I experimented with was the pipeline, which is in multi_step.py. For this setup, I had AI generate and defend bad fan fiction theories. The first agent was tasked with listing a few ideas. The second agent thought about the theories, and picked which one to do. The last agent generated text about the theory. It took some prompting to get the agents to work the way I wanted, doing theories vs fan fiction ideas. One of the problems I ran into was the models thinking they lived in the Star Wars universe, so when they were trying to 'prove' theories, they were trying to do things that would only work in-universe. One thing that was interesting in the multi-agent system was that if I wanted to change a prompt, I had potentially four places I could change it from.

I had a hard time deciding if this workflow was better or not. Because each piece was individual, the different pieces that got returned were more fully flushed out than just a single agent. However, it's hard to tell if the bad theories they generated were better in the multi agent system.

The next workflow I experimented was with the hub-and-spoke model. Instead of having the model decide delegation, I had pre-delegated tasks for travel planning. I had one model think about flights, one about hotels, and one about activities. I had the hub agent synthesize the information and return it to the user. I ran out of time to work with it, but one thing that would be interesting is giving the agents the opportunity to interact with each other. If a spoke agent wants to ask a question, how does it go about doing that? It's not as significant in the small problems I've been using, but I can see that letting each one do a small task will mean that each task will be done more reliably. I've never had problems with an agent forgetting instructions, but I've heard that it happens, so this would be a good way to fix it.
