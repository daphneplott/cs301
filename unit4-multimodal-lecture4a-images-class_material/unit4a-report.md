To see how the LLM can extract information from an image, I used the program from the demo in class. I then ran it on some of my saved images. One of the images was a paper assignment for a math class. It was able to describe that it was math, either notes or homeowkr, on a sheet of notebook paper. Another image was a picture I had drawn of a cafe. It was able to pretty accurately descirbe the different elements of my cafe, and comment on the artwork side, such as perspective and shading. Another image was an old photograph of the Hall of Mirrors during the signing of the treaty of Versailles. It could identify the elements of the image, the style of the hall, but wasn't able to classify it was the Hall of Mirrors. I then ran it on a screenshot of a LeetCode submission. It was able to correctly identify the elements in the submission, and the generated caption identified that it was a late night submission (around 9 pm). 

To try and see how well it identified certain pieces of the image, I then used the modify image to see if it could identify certain pieces to modify. One example I tried was seeing if it could circle Waldo. It would actually create a Waldo in the modified image. I then had it try to write up the name of a bakery from a picture where the name was displayed mirrored. It was able to pull it out really well. I then tried to get it to title my math homework based on the questions it could see. It had a hard time figuring out what the topic was. First, it was being too broad, then it was struggling when I gave it example topics (thinking it had to be one of those topics), but eventually I was able to get the right topic.
